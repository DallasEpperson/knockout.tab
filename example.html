Callback Tab Group

In some cases you will need to run some logic when a tab has changed, in this case you can use the onTabChanged callback

This will notify you when someone has clicked the tab in question so you can do some logic

Custom Show/Hide Logic Group

Not everyone wants their tab content to show and hide so simply, some people will want to transition the content in different ways

To do this you set the options 'onVisible' and 'onHidden', this will then use your custom logic to show and hide content rather than default showing and hiding

When hiding content it will be slid up

When showing content it will be slid down

Progmatically Showing Tabs

In certain cases, such as validation or other workflows you may need to show a tab from code

This is not nice as you need to know the tab selector in the code which makes the viewmodel dependant upon the view

However putting that aside as some times there is a need to do these sort of things, so you can use the 'ko.tabs.show(tabSelector)' method
